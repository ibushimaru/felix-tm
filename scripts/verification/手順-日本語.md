```
1. デモ動画を撮影 → YouTube に Unlisted でアップロード
```

### 撮影内容（≤2 分、英語ナレーション）

ナレーションは英語、ただし UI は日本語表示でも構いません。

| シーン | 時間 | 内容 |
|---|---|---|
| 1 | 0:00–0:05 | タイトルカード（白背景に「Felix TM — Translation Memory for Google Sheets — OAuth Verification Demo」） |
| 2 | 0:05–0:20 | `chrome://extensions/` で Felix TM が入ってる画面 → Google Sheets を開く → 拡張アイコンでサイドパネルを開く |
| 3 | 0:20–0:45 | **【重要】**サイドパネルの "Sign in with Google" をクリック → Google アカウント選択 → consent 画面 → ここで **URL バーに `accounts.google.com` が映るように一瞬止める**。consent 画面に「Felix TM がスプレッドシートにアクセス」とだけ表示されてることを確認できるように。Allow → サイドパネル "✓ Signed in" |
| 4 | 0:45–1:10 | A 列のセルをクリック → サイドパネルに即座に fuzzy match が出る（例: "Draw two cards." → "Draw a card." 75% / "Draw three cards." 80%）→ 別行に移動して即時更新を見せる |
| 5 | 1:10–1:30 | 80% マッチの "Set" ボタンをクリック → B 列に訳文が書き込まれる → "Auto Translate" で範囲一括処理 |
| 6 | 1:30–1:50 | Settings タブを開く → "Sign out" → DevTools → Application → IndexedDB → FelixTM の中身を見せる（TM がブラウザ内のみに保存されてることを示す） |
| 7 | 1:50–2:00 | クロージング（GitHub URL、privacy URL、連絡先 email） |

### ナレーション原稿（英語、コピペ用）

```
Scene 1: Felix TM is a Chrome extension that brings translation memory
into Google Sheets. This demo shows how it uses the spreadsheets OAuth
scope.

Scene 2: A translator opens a Google Sheet whose left column contains
source text and right column will hold translations. They open the
Felix TM side panel.

Scene 3: Sign-in opens Google's standard consent screen. Felix TM
requests exactly one scope: spreadsheets — to read source cells and
write target cells. No identity scope, no email, nothing else. The
consent dialog is served from accounts.google.com, as the URL bar
shows. After granting, the side panel shows the connected state.

Scene 4: As the translator moves between cells, Felix TM uses the
spreadsheets scope to read the source value of each active cell and
matches it against a translation memory stored locally in the browser.
Matches appear in real time. No data leaves the browser — matching
runs entirely in IndexedDB.

Scene 5: The translator confirms a match — Felix TM writes the
translation into the target cell using the same spreadsheets scope.
Auto Translate applies 100% matches across a selected range in one
action. These are the only situations where Felix TM writes — every
write is initiated by an explicit user click.

Scene 6: Sign-out revokes the OAuth token and clears it from Chrome's
identity cache. The translator's TM stays in their browser's
IndexedDB — Felix TM has no backend, so there is nothing to delete
server-side.

Scene 7: Felix TM source code, privacy policy, and contact information
are linked here. Thank you.
```

## 2. OAuth Consent Screen の Production 化

### URL

```
https://console.cloud.google.com/apis/credentials/consent?project=felix-tm
```

（プロジェクト ID は `429083959906`、表示名が違う場合は ID から選んでください）

### ステップ 1: Edit App で内容確認

下記の内容になっているか確認してください。違う場合は修正します。

| フィールド | 値 |
|---|---|
| App name | **Felix TM** |
| User support email | **ibushimaru@veriscio.com** |
| App logo | `plugins/chrome-extension/icons/icon128.png` をアップロード（120×120 以上の正方形 PNG） |
| Application home page | `https://ibushimaru.github.io/felix-tm/` |
| Application privacy policy link | `https://ibushimaru.github.io/felix-tm/privacy/` |
| Application terms of service link | （空欄 OK、または privacy 同じ URL） |
| Authorized domains | `ibushimaru.github.io` |
| Developer contact information | `ibushimaru@veriscio.com` |

→ **Save and continue**

### ステップ 2: Scopes の確認

リストにスコープが**1 つだけ**あることを確認:

- `https://www.googleapis.com/auth/spreadsheets` （Restricted）

もし他のスコープ（`userinfo.email` や `drive.file` 等）が残っていたら**削除してください**。
→ **Save**

### ステップ 3: Test users（そのまま）

ここはそのまま OK。Production 化後は意味を持ちません。

### ステップ 4: Publish App

OAuth consent screen の概要画面に戻り、**Publishing status: Testing** という表示の下にある **Publish App** ボタンをクリック → 確認ダイアログで OK。

ステータスが **In production — Verification required** に変わります。
（Restricted scope を使っているため必ず verification が必要）

### ステップ 5: Submit for verification

**Submit for verification** ボタンが出現するのでクリック → フォームが開きます:

| 項目 | 入力内容 |
|---|---|
| Application home page | `https://ibushimaru.github.io/felix-tm/` |
| Privacy policy URL | `https://ibushimaru.github.io/felix-tm/privacy/` |
| Authorized domains | `ibushimaru.github.io`（既に入っているはず） |
| **Demo video URL** | 先ほどアップロードした YouTube Unlisted の URL |
| **Justification for sensitive scopes** | `scope-justification.md` の "Sensitive scope justification" セクションをコピペ |
| **Justification for restricted scopes** | `scope-justification.md` の "Restricted scope justification" セクションをコピペ |
| **Limited Use attestation** | privacy policy の Limited Use セクションを引用、または `scope-justification.md` の最後のセクションをコピペ |
| **CASA assessment** | リンクが表示される → 別タブで CASA Tier 2 self-assessment を実施 → **Letter of Validation** という PDF を取得 → ここに添付 |

### ステップ 6: CASA Tier 2 self-assessment

これが一番ややこしい部分です。

1. Google が提供する CASA self-assessment のリンクをクリック（別サービス、Bishop Fox や Leviathan 等が運営）
2. アンケート形式で答えます。回答は `scripts/verification/casa-tier2-draft.md` に**英語で全部下書き済み**ですので、それをコピペしていけば OK
3. 完了すると Letter of Validation という PDF がメールで届く（即時〜数営業日）
4. PDF を Google の verification フォームに添付

### ステップ 7: 提出 → 待機

- 初動レスポンス: 3-5 営業日
- 合計レビュー期間: 4-8 週間（質問が 1-2 ラウンド来ることが多い）

**待機中の注意:**
- ❌ スコープを変更しない（タイマーがリセットされます）
- ❌ アプリ名を変更しない（同上）
- ✅ 拡張のコード修正は OK（マッチング、UI、バグ修正など、manifest のスコープを変えなければ）

## 3. CASA Tier 2 のリンクが見つからない場合

`scripts/verification/casa-tier2-draft.md` に詳細手順を書いてあります。要点:

- **CASA Tier 2** は Google が指定した第三者監査機関（Bishop Fox など）が実施
- 中規模アプリ向け（Tier 3 はより大規模なエンタープライズ向け）
- self-assessment（自己申告）でほとんど完結、ペネトレーションテスト不要
- 費用は無料（Google が指定の機関を用意）

verification フォームの中で「CASA assessment が必要」と出てきたタイミングで案内が出ます。
