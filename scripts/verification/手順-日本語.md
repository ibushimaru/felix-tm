# Felix TM — 公開準備 日本語ガイド

提出系のドキュメント本体（scope-justification.md、casa-tier2-draft.md 等）は
Google レビュアー（英語話者）が読む英語のままにしてあります。本書は
**作業手順だけ**を日本語でご案内するものです。

---

## 全体の流れ

```
1. デモ動画を撮影 → YouTube に Unlisted でアップロード
2. Cloud Console で OAuth consent screen を Production 化
3. Verification 提出フォームに記入（CASA Tier 2 含む）
4. 4-8 週間待つ（並行で Web Store 用ビルドと listing 作業を進める）
```

---

## 1. デモ動画の撮影

### 撮影前の準備

- [ ] Chrome のズームを 100% に
- [ ] ウィンドウサイズは 1920×1080 以上
- [ ] Chrome テーマはデフォルト（明るい色）
- [ ] **拡張機能が他に入っていない素のプロファイル**で録画
- [ ] サンプルの Google Sheet を事前に開く
  - A 列: 翻訳元（英語）10 行程度
  - B 列: 空（訳文を入れるところ）
- [ ] サンプル TM (`samples/sample-tm.tmx`) を Felix TM にインポート済み
- [ ] サンプル用語集もインポート済み
- [ ] **撮影前に Sign out しておく**（consent 画面を「ゼロから」見せるため）
- [ ] `chrome://identity-internals/` で Felix TM の token を revoke

### 撮影内容（≤2 分、字幕方式）

**ナレーションは不要**です。Google が求めるのは「視覚的に OAuth フローと
スコープ使用が確認できること」のみで、音声は要件外。各シーンに**英語の
テロップ（字幕オーバーレイ）**を 2-3 秒表示する方式が実務上最も多く、
レビュアーも停止して読めるため好まれます。

UI は日本語表示でも構いません（テロップで補足する形）。

| シーン | 時間 | 内容 | 画面に出すテロップ（英語） |
|---|---|---|---|
| 1 | 0:00–0:05 | タイトルカード | `Felix TM — OAuth Verification Demo` `Translation Memory for Google Sheets` |
| 2 | 0:05–0:20 | `chrome://extensions/` で Felix TM が入ってる画面 → Google Sheets を開く → 拡張アイコンでサイドパネルを開く | `Translator opens a Sheet and the Felix TM side panel.` |
| 3 | 0:20–0:45 | **【重要】**"Sign in with Google" をクリック → アカウント選択 → consent 画面で **URL バーに `accounts.google.com` が映るように一瞬止める**。consent 画面の「Felix TM がスプレッドシートにアクセス」表示が読めるように → Allow → "✓ Signed in" | `Sign in with Google.` `Single scope: spreadsheets.` `URL: accounts.google.com — standard Google consent.` |
| 4 | 0:45–1:10 | A 列のセルをクリック → サイドパネルに即座に fuzzy match が表示 → 別行に移動して即時更新を見せる | `Click a source cell → fuzzy matches appear in real time.` `All matching runs locally in IndexedDB.` |
| 5 | 1:10–1:30 | 80% マッチの "Set" ボタン → B 列に訳文書き込み → "Auto Translate" で範囲一括処理 | `Click "Set" → writes the translation to the target cell.` `Every write is user-initiated.` |
| 6 | 1:30–1:50 | Settings タブ → "Sign out" → DevTools → Application → IndexedDB → FelixTM の中身 | `Sign out revokes the OAuth token.` `TM data lives only in the browser's IndexedDB.` |
| 7 | 1:50–2:00 | クロージング | `github.com/ibushimaru/felix-tm` `ibushimaru.github.io/felix-tm/privacy/` `ibushimaru@veriscio.com` |

### テロップの作り方（最小限）

撮影ソフトの選択肢:

- **ScreenStudio / Loom** — テロップ機能あり、簡単
- **QuickTime Player（Mac 標準）+ iMovie** — 録画して iMovie でテロップ追加（無料）
- **OBS Studio** — リアルタイムでテロップ表示も可能

最低限のフォーマット:
- 黒背景 + 白文字、画面下 1/3 に配置
- フォントサイズ大きめ（読めるように）
- 表示時間 2-3 秒、フェードイン/アウト不要

### YouTube アップロード

1. YouTube Studio で **公開設定: 限定公開（Unlisted）**
2. タイトル: `Felix TM — OAuth Verification Demo`
3. 説明: 1 行の概要 + ホームページ URL + privacy URL
4. **アップロード後の URL を控える**（後で Google の verification フォームに貼ります）

## 2. OAuth Consent Screen の Production 化

### URL

```
https://console.cloud.google.com/apis/credentials/consent?project=felix-tm
```

（プロジェクト ID は `429083959906`、表示名が違う場合は ID から選んでください）

### ステップ 1: Edit App で内容確認

下記の内容になっているか確認してください。違う場合は修正します。

| フィールド | 値 |
|---|---|
| App name | **Felix TM** |
| User support email | **ibushimaru@veriscio.com** |
| App logo | `plugins/chrome-extension/icons/icon128.png` をアップロード（120×120 以上の正方形 PNG） |
| Application home page | `https://ibushimaru.github.io/felix-tm/` |
| Application privacy policy link | `https://ibushimaru.github.io/felix-tm/privacy/` |
| Application terms of service link | （空欄 OK、または privacy 同じ URL） |
| Authorized domains | `ibushimaru.github.io` |
| Developer contact information | `ibushimaru@veriscio.com` |

→ **Save and continue**

### ステップ 2: Scopes の確認

リストにスコープが**1 つだけ**あることを確認:

- `https://www.googleapis.com/auth/spreadsheets` （Restricted）

もし他のスコープ（`userinfo.email` や `drive.file` 等）が残っていたら**削除してください**。
→ **Save**

### ステップ 3: Test users（そのまま）

ここはそのまま OK。Production 化後は意味を持ちません。

### ステップ 4: Publish App

OAuth consent screen の概要画面に戻り、**Publishing status: Testing** という表示の下にある **Publish App** ボタンをクリック → 確認ダイアログで OK。

ステータスが **In production — Verification required** に変わります。
（Restricted scope を使っているため必ず verification が必要）

### ステップ 5: Submit for verification

**Submit for verification** ボタンが出現するのでクリック → フォームが開きます:

| 項目 | 入力内容 |
|---|---|
| Application home page | `https://ibushimaru.github.io/felix-tm/` |
| Privacy policy URL | `https://ibushimaru.github.io/felix-tm/privacy/` |
| Authorized domains | `ibushimaru.github.io`（既に入っているはず） |
| **Demo video URL** | 先ほどアップロードした YouTube Unlisted の URL |
| **Justification for sensitive scopes** | `scope-justification.md` の "Sensitive scope justification" セクションをコピペ |
| **Justification for restricted scopes** | `scope-justification.md` の "Restricted scope justification" セクションをコピペ |
| **Limited Use attestation** | privacy policy の Limited Use セクションを引用、または `scope-justification.md` の最後のセクションをコピペ |
| **CASA assessment** | リンクが表示される → 別タブで CASA Tier 2 self-assessment を実施 → **Letter of Validation** という PDF を取得 → ここに添付 |

### ステップ 6: CASA Tier 2 self-assessment

これが一番ややこしい部分です。

1. Google が提供する CASA self-assessment のリンクをクリック（別サービス、Bishop Fox や Leviathan 等が運営）
2. アンケート形式で答えます。回答は `scripts/verification/casa-tier2-draft.md` に**英語で全部下書き済み**ですので、それをコピペしていけば OK
3. 完了すると Letter of Validation という PDF がメールで届く（即時〜数営業日）
4. PDF を Google の verification フォームに添付

### ステップ 7: 提出 → 待機

- 初動レスポンス: 3-5 営業日
- 合計レビュー期間: 4-8 週間（質問が 1-2 ラウンド来ることが多い）

**待機中の注意:**
- ❌ スコープを変更しない（タイマーがリセットされます）
- ❌ アプリ名を変更しない（同上）
- ✅ 拡張のコード修正は OK（マッチング、UI、バグ修正など、manifest のスコープを変えなければ）

## 3. CASA Tier 2 のリンクが見つからない場合

`scripts/verification/casa-tier2-draft.md` に詳細手順を書いてあります。要点:

- **CASA Tier 2** は Google が指定した第三者監査機関（Bishop Fox など）が実施
- 中規模アプリ向け（Tier 3 はより大規模なエンタープライズ向け）
- self-assessment（自己申告）でほとんど完結、ペネトレーションテスト不要
- 費用は無料（Google が指定の機関を用意）

verification フォームの中で「CASA assessment が必要」と出てきたタイミングで案内が出ます。

## 困ったときは

- 「Demo video が ≤2 分に収まらない」 → Scene 4 を 15 秒に短縮、Scene 6 を 10 秒に短縮で OK
